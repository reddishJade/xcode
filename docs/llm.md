# Transformer LLM 工作原理：从文本表示到现代架构

## TL;DR

```
文本 → token → token ID → embedding → Transformer Blocks → 上下文化 embedding → Language Modeling Head → next token
```

LLM 的输出不是凭空"思考"出来的，而是在当前 context window 中，基于 token 表示、attention 关系和模型参数，**逐步预测下一个 token**。理解这一过程需要从语言的数值化表示出发，经过注意力机制的演进，最终到达现代 Transformer 的完整图景。

------

## 第一章：语言的数值化表示

在任何神经网络能够处理语言之前，文本必须先被转换为数字。这一挑战催生了从简单统计方法到语义感知嵌入的演进。

### 词袋模型（Bag of Words）

**词袋模型**是最早用于将文本转化为数值表示的方法之一。其基本流程是：首先将文本按空格切分为单词，即**分词**（tokenization）；然后构建一个覆盖所有出现单词的**词汇表**；最后统计输入中各单词在词汇表中的出现频率，生成归一化到 0 到 1 之间的**词袋向量**。

词袋模型的洞见在于，一个句子的含义不仅由它**包含**的词决定，也由它**不包含**的词传达。然而，这一方法存在根本性缺陷：它将语言视为纯粹的词频统计，完全忽视了词与词之间的**语义关系**和文本的深层含义。

### Word2Vec：捕捉语义的嵌入

2013 年发布的 **Word2Vec** 是最早成功捕捉文本语义的算法之一。它通过在海量文本（如整个维基百科）上训练**神经网络**，学习单词的语义表示，为每个词生成一个密集的**嵌入向量**（embedding）。

Word2Vec 的训练逻辑源自一个语言学直觉：**意义相近的词往往出现在相似的上下文中**。训练过程中，模型从语料中抽取词对，尝试预测两个词是否倾向于相邻出现。如果两个词共享大量相同的邻近词，它们的嵌入向量就会被拉近；反之则被推远。

以"cats"为例，其嵌入向量在"动物"和"复数"等维度得分较高，在"人类"和"水果"等维度得分较低。**嵌入向量的维度**代表某种隐含属性，虽然这些属性的具体含义是通过复杂数学学习而来、无法直接解读，但它们允许我们通过向量距离来比较词义的相似程度。维度数量通常超过 1000，每一维都是一个连续的数值。

Word2Vec 属于**表示模型**——其目标是将文本转换为有意义的数值，而非生成新文本。同样的逻辑可以通过对子词嵌入取平均来获得**词嵌入**，进而聚合为**句子嵌入**和**文档嵌入**。

### Word2Vec 的局限：静态嵌入

Word2Vec 的关键问题在于它生成的是**静态嵌入**：无论上下文如何，同一个词永远对应同一个向量。"bank"可以指银行，也可以指河堤，但 Word2Vec 无法区分。真正理解语言，需要能够随上下文动态调整的表示方式。

## 第二章：注意力机制的诞生

### 循环神经网络与编码-解码架构

为了捕捉文本的上下文，**循环神经网络**（RNN）被引入处理序列数据。在机器翻译等任务中，RNN 被组织为**编码器-解码器架构**：编码器将输入句子压缩为一个**上下文向量**，解码器则基于该向量逐步生成目标语言的输出。这一生成过程是**自回归**的——每一步都以所有已生成的词为输入，再预测下一个词。

然而，当输入句子很长时，**单一的上下文向量难以充分保留所有语义信息**，解码器在生成后期往往丢失重要的上下文。

### 注意力机制的解决方案

2014 年引入的**注意力机制**从根本上改进了这一架构。与其将整个输入压缩为一个向量，注意力机制允许解码器在生成每个词时，动态地**审视整个输入序列**，为每个输入词计算一个**注意力权重**，表示其与当前生成步骤的相关程度。

以翻译"I love llamas"为"Ik hou van lama's"为例：生成"Ik"时，模型主要关注"I"；生成"lama's"时，注意力自然转向"llamas"。高度相关的词对（如"I"和"Ik"）具有高注意力权重，而关联性弱的词对权重趋近于零。改进后的架构将每个输入词的**隐状态**（而非单一上下文向量）传递给解码器，使其获得更丰富的序列信息。

注意力机制的引入大幅提升了翻译质量，但 RNN 的**顺序处理本质**仍然制约了并行化能力——每个时间步依赖前一步的输出，无法充分利用现代 GPU 的并行算力。

## 第三章：Transformer 架构

### "Attention is All You Need"

2017 年的论文《Attention is All You Need》彻底抛弃了 RNN，提出了完全**基于注意力机制**的 **Transformer 架构**。这一设计的核心优势是：整个输入序列的所有位置可以**同时并行处理**，训练速度相比 RNN 大幅提升。

原始 Transformer 采用**编码器-解码器架构**，编码器负责表示输入序列，解码器负责生成输出序列。两者都由多个相同的块堆叠而成，每块包含**自注意力层**和**前馈神经网络层**。

### 自注意力（Self-Attention）

普通注意力机制比较的是两个不同序列；**自注意力**则将单个序列与自身比较，计算序列内部每个位置与其他所有位置之间的关联强度。这使每个 token 都能"感知"整个序列中其他 token 的存在，从而获得真正的上下文化表示。

以句子"the dog chased the llama because it"为例，模型需要判断"it"指代的是"dog"还是"llama"——这是 NLP 中的**共指消解**问题。自注意力在处理"it"时，能够审视前面所有 token，将"llama"的表示信息融入到"it"的向量中，确保后续生成语义一致。

### 掩蔽自注意力与解码器

解码器中使用**掩蔽自注意力**（Masked Self-Attention），其与普通自注意力的区别在于：通过移除注意力矩阵上三角部分的值，确保任意位置**只能关注其前面的 token**，防止在生成时"看到未来"，保证自回归生成的因果性。

解码器还包含一个**交叉注意力**层，同时处理已生成的内容和编码器输出的原始输入表示，实现编码信息向解码过程的传递。

### 架构的演进：BERT 与 GPT

原始编码器-解码器架构对翻译任务效果卓越，但并不便于适配其他任务。此后，两种专用变体主导了 NLP 领域：

**BERT**（2018）是**仅编码器**的架构，专注于生成高质量的上下文化文本表示。它在输入前添加特殊的 `[CLS]` token 作为整个序列的摘要表示，并通过**掩蔽语言建模**（随机遮盖部分词后让模型预测）进行预训练。BERT 非常适合分类、问答等理解型任务。

**GPT 系列**则是**仅解码器**的架构，专注于文本生成。解码器块使用掩蔽自注意力确保因果性，自回归地逐个预测下一个 token。这一范式随着参数规模的增长爆发出惊人的能力：GPT-1（1 亿参数，2018）→ GPT-2（10 亿参数，2019）→ GPT-3（1750 亿参数，2020）。2022-2023 年，ChatGPT（基于 GPT-3.5）的发布将大语言模型带入大众视野，专有模型与开源模型生态随之繁荣。

**当前主流**：绝大多数文本生成 LLM 都是**纯解码器模型**；编码器模型则继续用于文本嵌入、检索重排序等效率敏感的 NLP 任务。

## 第四章：分词器（Tokenizer）

### token 的本质

LLM 处理文本的第一步是**分词**：将文本切分为更小的片段，每个片段称为一个 **token**。token 不等同于"词"——它可以是完整单词、词的片段，甚至单个字符或字节。分词器拥有**固定词汇表**，遇到表外的词时，会通过**子 token 的组合**来表示。

### 四个粒度级别

分词存在从粗到细的四个级别，各有权衡：

**词级分词**将文本按空格切分为完整单词。概念简单，但词汇表必须覆盖所有可能的词形，对新词和罕见词无能为力。

**子词级分词**是当今主流选择。token 可以是完整词，也可以是词的片段——例如"vocalization"被切分为"vocal"和"ization"。这种方式在词汇表大小和表示能力之间取得良好平衡，即使遇到新词也能有效表示。

**字符级分词**将每个字符作为独立 token，词汇表极小（仅 26 字母加特殊符号），可表示任意语言，但需要更多 token 来编码相同的文本。

**字节级分词**是最细粒度的方案，用字节来编码字符，适合追求最大灵活性的场景。

### token 到嵌入的转换流程

分词之后，每个 token 对应一个**固定 ID**，再映射到一个**嵌入向量**（token embedding）。这些初始嵌入是**静态的**——它们独立于其他 token，仅代表该 token 的词汇身份。经过 Transformer 块的处理后，这些静态嵌入被转化为**上下文化嵌入**：每个位置的向量仍然对应原来的 token，但已融合了整个序列中其他 token 的信息。

### BERT vs GPT-4（Tiktoken）

以 BERT 分词器为例，它的词汇表约 30,000 个 token，处理"hello world!"时会生成：`[CLS]`、`hello`、`world`、`!`、`[SEP]`，其中 `[CLS]` 和 `[SEP]` 是用于分类和句子分隔的特殊 token。BERT 对大写字母处理较弱，遇到"Capitalization"这类词需要拆分为多个前缀带"#"的子 token；对完全未知的词则输出 `[UNK]` token。

GPT-4 使用的 **Tiktoken** 分词器词汇表约 100,000 个 token，是 BERT 的 3 倍以上，处理同样文本所需 token 数量更少，表示效率更高，且不包含 `[CLS]`、`[SEP]` 等面向理解任务的特殊 token。

词汇表规模的选择存在权衡：词汇表越大，稀有词的表示越精简，但需要学习和存储的嵌入向量也越多。

## 第五章：Transformer 块的内部结构

### 整体流程

分词后的 token 嵌入进入**Transformer 块堆栈**。每一层块接收上一层的输出向量，生成相同维度的新向量，最终由**语言建模头**（Language Modeling Head）将最后一层的输出转化为词表上的概率分布，从而预测下一个 token。

```
输入文本 → Tokenizer → [Block 1] → [Block 2] → ... → [Block N] → Language Modeling Head → 输出 token
```

每个 Transformer 块由两个主要组件构成：**自注意力层**负责融合上下文，**前馈神经网络层**负责知识存储与变换。

### 前馈神经网络（FFN）：知识的存储库

前馈层是模型中**学到的知识的物理体现**。直觉上可以这样理解：如果 Transformer 块中只有前馈层而没有注意力，它仍然能够生成补全——例如处理"Shawshank"时，前馈层会预测"redemption"，因为这一词对在训练数据中高频共现。

前馈层的典型结构为"扩展-收缩"模式：输入向量经过一个**扩展层**升维（维度增大数倍），在高维空间中完成非线性变换，再经过**收缩层**降回原始维度。这些密集连接中存储的权重，编码了模型从海量数据中提炼的语言规律、世界知识和推理模式。

### 自注意力层：上下文的整合者

自注意力层的职责是**将序列中其他位置的相关信息融入当前 token 的表示**。其计算可以分解为两个步骤：

**第一步，相关性评分**：通过**查询（Query）矩阵**和**键（Key）矩阵**的乘积，计算当前 token 与序列中每个其他 token 之间的关联强度，生成一组归一化的注意力权重（所有权重之和为 100%）。

**第二步，信息融合**：将每个 token 的**值（Value）向量**按照上一步得到的注意力权重进行加权求和，得到当前位置融合了上下文信息的新向量表示。

这三个投影矩阵——**Query、Key、Value**——的权重是模型通过训练学习得到的，赋予了自注意力动态感知上下文的能力。

### 多头注意力（Multi-Head Attention）

实际中，自注意力并非只运行一次，而是在多个**注意力头**（attention head）中**并行进行**。每个头拥有独立的 Query、Key、Value 权重矩阵，可能学到不同的注意力模式——例如一个头关注语法关系，另一个头捕捉长距离语义依赖。所有头的输出最终被**拼接并融合**，形成自注意力层的最终输出。

## 第六章：推理过程与 KV 缓存

### 逐 token 生成的迭代过程

理解 LLM 推理的关键直觉是：**模型逐个生成 token**。生成第一个 token 时，完整的输入提示被并行送入所有 Transformer 块；生成第二个 token 时，"输入提示 + 第一个生成 token"被整体重新输入模型；以此类推，每一步都将已生成的内容追加到上下文中。

```
第一步：[输入提示] → Transformer → [Token 1]
第二步：[输入提示 + Token 1] → Transformer → [Token 2]
第三步：[输入提示 + Token 1 + Token 2] → Transformer → [Token 3]
...
```

### KV 缓存：消除重复计算

上述过程存在明显的效率问题：每生成一个新 token，前面所有位置的注意力计算都需要重复执行。**KV 缓存**（KV Cache）通过缓存每个 token 对应的 Key 和 Value 向量来解决这一问题——某个位置一旦被计算过，后续步骤直接复用缓存结果，无需重算。KV 缓存是**使 Transformer 推理更快的最关键优化之一**，使得流式对话成为可能。

### 解码策略与温度

语言建模头生成词表上的概率分布后，需要通过**解码策略**从中选出实际输出的 token。

**贪心解码**（Greedy Decoding）始终选择概率最高的 token，对应**温度参数为 0** 的设置，输出确定且重复。**Top-P 采样**则从概率分布的累积 top-P 范围内随机采样，**温度大于 0** 时引入随机性，使模型有时选择非最优 token，输出因而更自然多样。这就是为什么同一提示多次输入 ChatGPT 会得到不同回答——本质上是解码策略和温度共同作用的结果。

### 关键性能指标：TTFT

**首 token 延迟**（Time To First Token, TTFT）衡量模型处理完整输入并生成第一个 token 所需的时间，是用户感知响应速度的核心指标。第一个 token 生成后，由于 KV 缓存的加速，后续 token 的生成通常更快。

## 第七章：高效注意力机制

自注意力通常是 Transformer 中**计算量最密集的部分**。随着上下文长度增长，标准多头注意力的计算开销呈二次方增长，研究者因此提出了多种优化方案。

### 分组查询注意力（Grouped Query Attention）

标准多头注意力中，每个注意力头都拥有独立的 Key 和 Value 矩阵，参数量庞大。**多查询注意力**（Multi-Query Attention）让所有头共享同一套 Key 和 Value，大幅降低参数量和计算成本，但可能损失部分表示能力。**分组查询注意力**（Grouped Query Attention）是两者之间的折中方案：将注意力头分成若干组，同组内共享 Key 和 Value 矩阵——既比标准多头注意力更高效，又比多查询注意力表达能力更强，是当前大型模型的主流选择。

以 Llama 3.1 8B 为例，其拥有 32 个查询头但仅 8 个键值组，每组服务 4 个查询头。

### 稀疏注意力（Sparse Attention）

标准注意力允许每个 token 关注所有前面的 token，当序列极长时开销过大。**稀疏注意力**并非在每一层都执行完整注意力，而是间隔使用局部注意力模式：例如**步长模式**（每个位置只看最近几个 token，并跳跃看特定位置）和**固定模式**（只看上下文中特定区间内的 token）。稀疏与完整注意力层交替出现，在保持性能的同时大幅降低计算量。

### 环形注意力（Ring Attention）

为处理 100,000 甚至 1,000,000 token 级别的超长上下文，**环形注意力**将序列分布到多个设备上并行计算，每个设备处理局部的 Key/Value，计算结果在设备间以环形方式传递，实现超大上下文窗口的高效注意力计算。

## 第八章：现代 Transformer 的关键改进

原始 Transformer（2017）与当代主流模型在整体思路上高度一致，但在几个关键细节上有所演进。

### 位置编码的演变

Transformer 的计算本质上是无序的——如果不加入位置信息，序列中词的顺序对模型而言毫无意义（退化为词袋）。**位置编码**的作用正是向模型传达每个 token 在序列中的位置。

原始 Transformer 在输入的最开始，通过**正弦/余弦函数组合**将绝对位置信息叠加到 token 嵌入上。这种**静态位置编码**简单有效，但对多文档打包训练不够友好。

现代高效训练实践中，往往将多个短文档拼接在一个序列槽内，最大限度利用计算资源。此时，静态位置编码会错误地将不同文档的 token 位置混淆，因为它无法感知文档边界。

**旋转位置嵌入**（Rotary Position Embedding, **RoPE**）是当前的主流解决方案。RoPE 的位置信息**不在模型开头添加，而是在每个自注意力层内部、相关性评分步骤之前**，直接作用于 Query 和 Key 向量：通过特定的旋转变换，将位置信息编码为向量旋转角度，使得注意力分数天然反映 token 之间的**相对位置**关系。这一设计既支持灵活的文档打包，又允许在每层动态调整位置感知，是 Llama、Mistral 等系列模型的标配。

### Pre-LN 与残差连接

原始 Transformer 在自注意力和前馈层**之后**进行层归一化（Post-LN）；现代模型将层归一化移到这两个子层**之前**（Pre-LN），训练更加稳定，实验表明模型性能更好。**残差连接**贯穿每个子层，将子层的输入直接加回输出，为梯度提供直接传播路径，有效缓解深层网络的梯度消失问题。

## 第九章：混合专家模型（Mixture of Experts）

### 核心思想

**混合专家模型**（MoE）是扩展 Transformer 能力的重要架构变体。在标准 Transformer 中，每个前馈层是单一的密集神经网络，对所有输入一视同仁。MoE 将前馈层替换为**多个专家**（每个专家是一个独立的前馈神经网络）加上一个**路由器**（Router）的组合。

**路由器**本身是一个轻量级分类器，对每个输入 token 计算各专家的概率分数，据此将 token 路由到最合适的一个（或几个）专家进行处理。其余专家在该 token 的处理过程中保持静默。这种**稀疏激活**机制是 MoE 的核心——模型加载了所有专家的参数，但推理时每个 token 只激活其中一小部分。

### 专家的分工

MoE 中的专家**并非按学科领域专门化**（不存在"历史专家"或"代码专家"），而是在 token 级别学习**句法和语言特征**的处理偏好——例如某个专家擅长处理标点符号，另一个专家专注于动词，第三个专家处理连接词。不同层的专家集合彼此独立，路由器在每层独立作出路由决策。

### 稀疏参数 vs. 活跃参数

以 **Mixtral 8x7B** 为例：模型共有约 46B 参数需要加载，但推理时路由器通常只激活 2 个专家，实际参与计算的**活跃参数**约为 29B。这意味着以相对较高的**内存成本**换取了显著降低的**计算开销**，在生产环境中极具效率优势。

MoE 的主要优点是在扩大参数规模的同时控制计算量，且专家的分工有助于减少冗余。其主要挑战包括内存需求高、可能出现某些专家被过度使用而其他专家被冷落的**负载不均衡**问题，以及更复杂的训练策略。

MoE 并不局限于 Transformer——它本质上是对前馈层的改造，因此同样可以应用于 Mamba 等**状态空间模型**。

## 第十章：RAG 与 Agent

即使是最大规模的 LLM，其知识也受限于训练数据的截止时间，且 context window 有限。**检索增强生成**（Retrieval-Augmented Generation, **RAG**）和 **Agent** 架构为此提供了两种互补的扩展方式。

### RAG：把外部文档引入上下文

RAG 的核心思路是：**在生成之前，先从外部知识库中检索与查询相关的文档片段，将其注入 context window，再由 LLM 基于这些文档生成答案**。

典型流程如下：用户查询首先被转化为嵌入向量，然后在预先构建的文档向量库（通常使用编码器模型生成文档嵌入）中进行相似度检索，取出最相关的若干段落，拼接到提示中一并输入 LLM。LLM 的生成因此有据可查，能够引用最新的外部信息，大幅降低"幻觉"风险。

RAG 适合以下场景：知识库频繁更新（如企业内部文档、实时新闻）、回答需要引用来源、或信息量超过单次 context window 容量。

### Agent：让 LLM 主动决策与行动

**Agent** 将 LLM 从被动的文本生成器提升为**能够规划和执行多步任务的主动决策者**。Agent 的关键能力在于：LLM 可以根据当前目标，自主决定是否需要调用外部工具（如搜索引擎、代码执行器、数据库、API），观察工具返回的结果，将结果纳入上下文，再决定下一步行动，循环迭代直到任务完成。

与 RAG 相比，Agent 并非预先检索再一次性生成，而是在**推理过程中动态决定何时检索、检索什么、如何使用结果**。工具调用和中间结果不断追加到 context 中，形成一个自我完善的推理链条。

RAG 和 Agent 并非互斥——在 Agent 系统中，检索工具本身就是众多可调用工具之一。当任务需要结合实时信息、外部系统操作和多步推理时，两者往往协同工作，构成完整的 LLM 应用架构。

## 附录：理解模型架构参数

掌握了以上概念后，你已经可以直接阅读和解读模型论文中的架构规格表。以 **Llama 3.1 8B** 为例：

| 参数                   | 值      | 含义                       |
| ---------------------- | ------- | -------------------------- |
| Layers                 | 32      | 堆叠 32 个 Transformer 块  |
| Model Dimension        | 4096    | 所有向量的嵌入维度         |
| Feed-Forward Dimension | 14336   | 前馈层扩展维度             |
| Attention Heads        | 32      | 查询头数量                 |
| Key-Value Heads        | 8       | 键值组数（分组查询注意力） |
| Vocabulary Size        | 128,000 | 分词器词汇表大小           |
| Positional Embeddings  | RoPE    | 旋转位置嵌入               |

32 个查询头配合 8 个键值组，意味着每组服务 4 个查询头，这是分组查询注意力的典型配置。词汇表 128,000 个 token 对应的嵌入矩阵大小为 128,000 × 4,096，这部分参数本身就相当可观。理解每个数字背后的架构含义，是深入研究和复现当代 LLM 的必要基础。
